## Overview

The app provides two barcode scanning modes using the `mobile_scanner` package:

1. **Continuous Scanning** (Home Page): Always-on camera for rapid product additions
2. **Single Scan Mode** (Product Management): One-time scan that returns the barcode value

## Scanner Modes

### Continuous Scanner (Home Page)

Used for billing operations with:
- `DetectionSpeed.normal`: Balanced speed and accuracy
- **Cooldown mechanism**: 2-second delay between identical barcodes
- **Auto-add to cart**: Products automatically added on detection
- **Error handling**: Shows snackbar for unknown products

```dart lib/features/billing/presentation/pages/home_page.dart
final MobileScannerController _scannerController = MobileScannerController(
  detectionSpeed: DetectionSpeed.normal,
  returnImage: false,
);
```

### Single Scan Mode (Product Management)

Used for product operations with:
- `DetectionSpeed.noDuplicates`: Scans once and returns
- **Auto-close**: Scanner closes after successful scan
- **Return value**: Barcode string passed back to calling screen

```dart lib/features/billing/presentation/pages/scanner_page.dart
final MobileScannerController controller = MobileScannerController(
  detectionSpeed: DetectionSpeed.noDuplicates,
  returnImage: false,
);
```

## Barcode Detection Flow

### Continuous Mode Detection

<Steps>
  <Step title="Camera Captures Frame">
    MobileScanner continuously processes camera frames
  </Step>
  
  <Step title="Barcode Detected">
    When barcode is recognized, `onDetect` callback fires
  </Step>
  
  <Step title="Cooldown Check">
    System checks if same barcode was scanned within last 2 seconds
  </Step>
  
  <Step title="Vibration Feedback">
    Device vibrates if vibration hardware is available
  </Step>
  
  <Step title="Add to Cart">
    BillingBloc receives `ScanBarcodeEvent` and looks up product
  </Step>
  
  <Step title="Update UI">
    Cart updates with new item or incremented quantity
  </Step>
</Steps>

### Single Scan Detection

<Steps>
  <Step title="Open Scanner">
    User taps QR icon from product add/edit page
  </Step>
  
  <Step title="Align Barcode">
    User positions barcode within frame
  </Step>
  
  <Step title="Auto-Detect">
    Scanner detects barcode and vibrates
  </Step>
  
  <Step title="Return Value">
    Scanner closes and returns barcode string via `context.pop(barcode.rawValue)`
  </Step>
  
  <Step title="Pre-fill Form">
    Calling page receives barcode and fills form field
  </Step>
</Steps>

## Implementation Details

### Detection Handler with Cooldown

```dart lib/features/billing/presentation/pages/home_page.dart
// Cooldown mapping to prevent rapid firing of the same barcode
final Map<String, DateTime> _lastScanTimes = {};

void _onDetect(BarcodeCapture capture) async {
  final List<Barcode> barcodes = capture.barcodes;
  final now = DateTime.now();

  for (final barcode in barcodes) {
    if (barcode.rawValue != null) {
      final rawValue = barcode.rawValue!;

      // Cooldown logic: 2 seconds per identical barcode
      if (_lastScanTimes.containsKey(rawValue)) {
        final lastScan = _lastScanTimes[rawValue]!;
        if (now.difference(lastScan).inSeconds < 2) {
          continue;
        }
      }

      _lastScanTimes[rawValue] = now;

      // Vibrate
      final hasVibrator = await Vibration.hasVibrator();
      if (hasVibrator == true) {
        Vibration.vibrate();
      }

      if (mounted) {
        context.read<BillingBloc>().add(ScanBarcodeEvent(rawValue));
      }
      break; // Process one barcode at a time per frame
    }
  }
}
```

### Single Scan Handler

```dart lib/features/billing/presentation/pages/scanner_page.dart
void _onDetect(BarcodeCapture capture) async {
  if (_isScanned) return;
  final List<Barcode> barcodes = capture.barcodes;

  for (final barcode in barcodes) {
    if (barcode.rawValue != null) {
      _isScanned = true;
      
      // Vibrate
      final hasVibrator = await Vibration.hasVibrator();
      if (hasVibrator == true) {
        Vibration.vibrate();
      }

      if (mounted) {
        context.pop(barcode.rawValue);
      }
      break; // Only take first one
    }
  }
}
```

## Permissions

The app requires camera permission to function. On first launch:

1. App requests camera permission
2. User grants or denies access
3. If denied, scanner shows error state
4. User can grant permission later in system settings

## Supported Formats

MobileScanner supports all common formats:

- **1D Barcodes**: EAN-8, EAN-13, UPC-A, UPC-E, Code 39, Code 93, Code 128, ITF, Codabar
- **2D Codes**: QR Code, Data Matrix, Aztec, PDF417

## Performance Optimization

  <Accordion title="returnImage: false">

  </Accordion>
  
  <Accordion title="DetectionSpeed Settings">
    - `normal`: Balanced for continuous scanning
    - `noDuplicates`: Prevents multiple scans of same code
  </Accordion>

    2-second cooldown prevents accidental duplicate cart additions

</AccordionGroup>

## Troubleshooting

<Warning>
  If scanning is slow or unresponsive:
  1. Ensure good lighting or enable flash
  2. Clean camera lens
  3. Hold barcode steady within frame
  4. Try different angles if barcode is glossy
</Warning>

## Best Practices

<Check>Use proper lighting - natural light or flash in dim environments</Check>
<Check>Hold device steady at 6-12 inches from barcode</Check>
<Check>Ensure barcode is flat and not wrinkled or damaged</Check>
<Check>Align barcode within the green frame guide</Check>
<Check>Wait for vibration feedback before moving to next product</Check>
