## Cart Data Model

Cart items combine product data with quantity:

```dart lib/features/billing/domain/entities/cart_item.dart
class CartItem extends Equatable {
  final Product product;
  final int quantity;

  const CartItem({
    required this.product,
    this.quantity = 1,
  });

  double get total => product.price * quantity;
}
```

## Home Page Interface

The home page is split into two sections:

### Top Section (40% height)

### Bottom Section (60% height + overlap)

## User Workflows

### Scanning Products

<Steps>
  <Step title="Ensure Camera is On">
    The camera starts automatically on the home page. If off, tap the camera icon to turn it on
  </Step>
  
  <Step title="Align Barcode">
    Position the product barcode within the green frame
  </Step>
  
  <Step title="Automatic Detection">
    The app automatically:
    - Detects the barcode
    - Looks up the product in the database
    - Adds it to the cart with vibration feedback
    - Shows error if product not found
  </Step>
  
  <Step title="Scan Cooldown">
    Each barcode has a 2-second cooldown to prevent duplicate scans
  </Step>
</Steps>

<Note>
  The scanner uses `DetectionSpeed.normal` and implements cooldown logic to prevent rapid duplicate additions.
</Note>

### Managing Cart Items

<Steps>
  <Step title="View Cart">
    Scanned items appear immediately in the bottom panel
  </Step>
  
  <Step title="Adjust Quantity">
    Use the +/- buttons to change quantity:
    - **+**: Increment quantity
    - **-**: Decrement quantity (removes if quantity becomes 0)
  </Step>
  
  <Step title="Monitor Total">
    The total price updates automatically as you modify quantities
  </Step>
</Steps>

### Checkout Process

<Steps>
  <Step title="Review Order">
    Tap the "Review Order" button at the bottom
    
    <Note>Button is disabled when cart is empty</Note>
  </Step>
  
  <Step title="Verify Items">
    The checkout page shows:
    - Product table with quantities, prices, and totals
    - Grand total at the bottom
    - UPI payment QR code (if configured)
  </Step>
  
  <Step title="Accept Payment">
    Customer scans the UPI QR code to pay
    
    The QR code contains:
    ```
    upi://pay?pa={upiId}&pn={shopName}&am={totalAmount}&cu=INR
    ```
  </Step>
  
  <Step title="Print Receipt">
    Tap "Print Receipt" to send receipt to thermal printer
  </Step>
  
  <Step title="Complete Transaction">
    Tap back to clear the cart and return to scanning
  </Step>
</Steps>

## Implementation Details

### Barcode Detection with Cooldown

```dart lib/features/billing/presentation/pages/home_page.dart
void _onDetect(BarcodeCapture capture) async {
  final List<Barcode> barcodes = capture.barcodes;
  final now = DateTime.now();

  for (final barcode in barcodes) {
    if (barcode.rawValue != null) {
      final rawValue = barcode.rawValue!;

      // Cooldown logic: 2 seconds per identical barcode
      if (_lastScanTimes.containsKey(rawValue)) {
        final lastScan = _lastScanTimes[rawValue]!;
        if (now.difference(lastScan).inSeconds < 2) {
          continue;
        }
      }

      _lastScanTimes[rawValue] = now;

      // Vibrate
      final hasVibrator = await Vibration.hasVibrator();
      if (hasVibrator == true) {
        Vibration.vibrate();
      }

      if (mounted) {
        context.read<BillingBloc>().add(ScanBarcodeEvent(rawValue));
      }
      break;
    }
  }
}
```

### Cart State Management

```dart lib/features/billing/presentation/bloc/billing_bloc.dart
void _onAddProductToCart(
    AddProductToCartEvent event, Emitter<BillingState> emit) {
  // Clear error when adding
  final cleanState = state.copyWith(error: null);

  final existingIndex = cleanState.cartItems
      .indexWhere((item) => item.product.id == event.product.id);
  if (existingIndex >= 0) {
    // Increment quantity if product already in cart
    final existingItem = cleanState.cartItems[existingIndex];
    final updatedItems = List<CartItem>.from(cleanState.cartItems);
    updatedItems[existingIndex] =
        existingItem.copyWith(quantity: existingItem.quantity + 1);
    emit(cleanState.copyWith(cartItems: updatedItems, error: null));
  } else {
    // Add new item with quantity 1
    final newItem = CartItem(product: event.product);
    emit(cleanState.copyWith(
        cartItems: [...cleanState.cartItems, newItem], error: null));
  }
}
```

## Billing Events

The BillingBloc handles these events:

| Event | Description |
|-------|-------------|
| `ScanBarcodeEvent` | Looks up product by barcode and adds to cart |
| `AddProductToCartEvent` | Adds product or increments quantity |
| `RemoveProductFromCartEvent` | Removes item from cart |
| `UpdateQuantityEvent` | Changes item quantity |
| `ClearCartEvent` | Empties the entire cart |
| `PrintReceiptEvent` | Prints receipt to thermal printer |

## Error Handling

The app handles these error scenarios:

- **Product Not Found**: Shows red snackbar with barcode value
- **Cart Empty**: Disables checkout button
- **Printer Not Connected**: Shows error when attempting to print
- **UPI Not Configured**: Hides QR code section
