## Shop Data Model

```dart lib/features/shop/domain/entities/shop.dart
class Shop extends Equatable {
  final String name;           // Shop name
  final String addressLine1;   // Primary address
  final String addressLine2;   // Secondary address (optional)
  final String phoneNumber;    // Contact number
  final String upiId;          // UPI ID for payments
  final String footerText;     // Receipt footer message

  const Shop({
    this.name = '',
    this.addressLine1 = '',
    this.addressLine2 = '',
    this.phoneNumber = '',
    this.upiId = '',
    this.footerText = '',
  });
}
```

### Form Fields

#### Shop Name (Required)
- **Field**: Text input
- **Example**: "QuickMart Superstore"
- **Usage**: 
  - Receipt header (large, bold, centered)
  - UPI QR code payee name
  - Settings page profile display

#### Address Line 1 (Required)
- **Field**: Text input
- **Example**: "Samrajpet, Mecheri"
- **Usage**: Primary address on receipt

#### Address Line 2 (Optional)
- **Field**: Text input
- **Example**: "Salem - 636453"
- **Usage**: Secondary address line on receipt

#### Phone Number (Required)
- **Field**: Phone input
- **Example**: "+91 7010674588"
- **Usage**: Contact info on receipt

#### UPI ID (Optional)
- **Field**: Text input
- **Example**: "dineshsowndar@oksbi"
- **Format**: `username@bankcode`
- **Usage**: Generates UPI payment QR code on checkout

<Note>
If UPI ID is not provided, the payment QR code section will be hidden on the checkout page.
</Note>

#### Receipt Footer Text (Optional)
- **Field**: Multiline text (max 60 characters)
- **Example**: "Thank you, Visit again!!!"
- **Usage**: Centered message at bottom of receipt

## Shop Details Form Implementation

```dart lib/features/shop/presentation/pages/shop_details_page.dart
void _saveShop() {
  if (_formKey.currentState!.validate()) {
    final shop = Shop(
      name: _nameController.text,
      addressLine1: _address1Controller.text,
      addressLine2: _address2Controller.text,
      phoneNumber: _phoneController.text,
      upiId: _upiController.text,
      footerText: _footerController.text,
    );

    context.read<ShopBloc>().add(UpdateShopEvent(shop));
  }
}
```

## State Management

Shop data is managed using BLoC:

### Events

| Event | Description |
|-------|-------------|
| `LoadShopEvent` | Load shop details from Hive |
| `UpdateShopEvent` | Save/update shop information |

### States

| State | Description |
|-------|-------------|
| `ShopLoading` | Loading shop data |
| `ShopLoaded` | Shop data loaded successfully |
| `ShopOperationSuccess` | Save operation completed |
| `ShopError` | Error occurred |

### Profile Display

Shop name generates initials for avatar:

```dart
String shopName = 'Elite Groceries';
String initials = 'EG';
if (state is ShopLoaded && state.shop.name.isNotEmpty) {
  shopName = state.shop.name;
  final parts = shopName.split(' ');
  initials = parts
      .take(2)
      .map((p) => p.isNotEmpty ? p[0].toUpperCase() : '')
      .join('');
  if (initials.isEmpty) initials = 'S';
}
```

## Data Persistence

Shop settings are stored in Hive database:

- **Box Name**: Usually a dedicated shop settings box
- **Storage**: Local device storage
- **Auto-load**: Settings load on app startup
- **Sync**: No cloud sync (local only)

## Validation Rules

<AccordionGroup>
  <Accordion title="Shop Name">

    - No length limit
    - Capitalized words
    - Used in multiple places (receipts, QR codes, profile)
  </Accordion>
  
  <Accordion title="Address Lines">
    - Address Line 1: Required
    - Address Line 2: Optional

    - Required field
    - Phone keyboard type
    - No format validation (accepts any format)
    - Include country code for international
  </Accordion>
  
  <Accordion title="UPI ID">
    - Optional
    - Format: `username@bankcode`
    - No validation enforced
    - Controls QR code visibility
  </Accordion>

## Troubleshooting

<Warning>
  **UPI QR Code Not Showing**
  
  Ensure UPI ID field is filled in Shop Details. Empty UPI ID hides the QR code section.
</Warning>

<Warning>
  **Receipt Missing Information**
  
  If receipt prints with missing shop details, verify all fields are saved in Shop Details page.
</Warning>
