## What is Flutter Billing App?

Flutter Billing App is a feature-rich, high-performance offline-first billing and Point of Sale (POS) application built with Flutter. Designed for seamless retail checkout operations, it features barcode scanning, thermal Bluetooth printing, and robust local data persistence.

This application serves as a complete offline POS system for small to medium-sized retail shops, streamlining the checkout process, catalog management, and receipt generation entirely on-device.

## Technology Stack

Built leveraging industry-standard architectural principles ensuring scalability, separation of concerns, and robust testability:

- **Framework**: [Flutter](https://flutter.dev/) (SDK >=3.1.0)
- **State Management**: `flutter_bloc` with BLoC pattern
- **Dependency Injection**: `get_it` service locator
- **Routing**: `go_router` for declarative navigation
- **Local Database**: `hive` & `hive_flutter` for offline storage
- **Functional Programming**: `fpdart` for error handling
- **Hardware Integrations**: `mobile_scanner` (barcodes), `print_bluetooth_thermal`

## Use Cases

<AccordionGroup>
  <Accordion title="Rapid Billing Entry">
    A cashier launches the app, navigates to the checkout page, and uses the device camera to instantly scan product barcodes. Products are added to the cart, the total is calculated including taxes, and a receipt is finalized.
  </Accordion>
  <Accordion title="Physical Receipt Generation">
    After checkout confirmation, the app triggers a connected external Bluetooth thermal POS printer to instantly print an itemized paper receipt with the shop's header.
  </Accordion>
  <Accordion title="Inventory Management">
    A manager opens the Product feature to add new stock to the local database, taking a picture of the barcode to bind the SKU for future lightning-fast checkouts.
  </Accordion>
  <Accordion title="No-Connection Operation">
    The business operates a stall at an exhibition with poor networking. The app functions entirely via its embedded Hive local database and Bluetooth, completely undisturbed by network drops.
  </Accordion>
</AccordionGroup>

## Architecture

Flutter Billing App follows **Clean Architecture** principles with a **Feature-First** structure:

```
lib/
├── core/                    # Shared utilities and components
│   ├── data/               # Global data sources (Hive initialization)
│   ├── error/              # Standardized Failure/Exception models
│   ├── theme/              # UI aesthetics, typography, styling
│   ├── usecase/            # Base UseCase contracts
│   ├── utils/              # Helpers (PrinterHelper, formatters)
│   └── widgets/            # Reusable UI widgets
│
└── features/               # Independent feature modules
    ├── billing/            # Cart, Checkout, Invoice Generation
    ├── product/            # Inventory management
    ├── settings/           # Printer connections, App settings
    └── shop/               # Shop details configuration
```

Each feature is subdivided into Clean Architecture layers: `data`, `domain`, and `presentation`.

## Quick Start

<Steps>
  <Step title="Install Flutter SDK">
    Ensure you have Flutter SDK ^3.1.0 or higher installed on your machine.
  </Step>
  <Step title="Clone and Install">
    Clone the repository and install dependencies:
    ```bash
    git clone https://github.com/Dinesh-Sowndar/flutter_billing_app
    cd flutter_billing_app
    flutter pub get
    ```
  </Step>
  <Step title="Generate Code">
    Run code generation for Hive adapters and JSON serialization:
    ```bash
    dart run build_runner build --delete-conflicting-outputs
    ```
  </Step>
  <Step title="Run the App">
    Launch the app on your device or emulator:
    ```bash
    flutter run
    ```
  </Step>
</Steps>
